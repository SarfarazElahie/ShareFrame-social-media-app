🔄 Data Flow Diagram
USER REQUEST
    ↓
[POST /create-post] ← Image + Caption
    ↓
[Multer Memory Storage] ← File buffered in RAM
    ↓
[uploadFile() Service] ← Sends buffer to cloud storage
    ↓
[Cloud Storage] → Returns image URL
    ↓
[MongoDB Create Post] ← Saves {image: url, caption, createdAt}
    ↓
[JSON Response] → {message, post object}

GET REQUEST
    ↓
[GET /posts]
    ↓
[MongoDB Find All] ← Fetches all posts
    ↓
[JSON Response] → {message, posts array}

📄 File-by-File Breakdown
1. server.js (Entry Point - Root Level)
Purpose: Start the application
What it does:
├── Loads environment variables (.env file)
├── Imports Express app from src/app.js
├── Connects to MongoDB via connectDB()
└── Listens on PORT 3000

Critical Details:
- Must run BEFORE app.listen() → connectDB() called synchronously
- This async function runs in background (no await)
- Server starts immediately even if DB connection incomplete
- ⚠️ ISSUE: No error handling if DB connection fails

2. db.js (Database Connection)
Location: src/db/db.js
Purpose: Establish MongoDB connection
Dependencies:
├── mongoose (MongoDB ODM library)
└── process.env.MONGO_URI (from .env file)

Function: connectDB()
├── Type: Async function
├── Action: mongoose.connect(process.env.MONGO_URI)
├── Logs: "Connected to DB..." on success
└── Returns: Promise

Critical Details:
- Requires MONGO_URI in .env (MongoDB connection string)
- No error handling → if connection fails, silent failure
- Called in server.js but NOT awaited
- Should be: await connectDB() or add .catch() handler
- ⚠️ ISSUE: Server might start before DB is ready

3. post.model.js (Database Schema)
Location: src/models/post.model.js
Purpose: Define Post structure
Schema Fields:
┌─────────────┬──────────┬──────────────────────────────────┐
│ Field       │ Type     │ Details                          │
├─────────────┼──────────┼──────────────────────────────────┤
│ image       │ String   │ REQUIRED - Cloud storage URL     │
│ caption     │ String   │ REQUIRED - Post text             │
│             │          │ Default: "" (empty string)       │
│ createdAt   │ Date     │ AUTO - Timestamp when created    │
│             │          │ Default: Current date/time       │
└─────────────┴──────────┴──────────────────────────────────┘

MongoDB Collection Name: "post" (singular)

Example Document:
{
    _id: ObjectId("..."),
    image: "https://cloud-storage.com/image123.jpg",
    caption: "Beautiful sunset! 🌅",
    createdAt: ISODate("2024-01-15T10:30:00Z")
}

Critical Details:
- image URL stored as string (not binary)
- All fields except _id are user-provided or auto-generated
- Default createdAt = Date.now (server time when post created)


4. app.js (Express Routes & Controllers)
Location: src/app.js
Purpose: Define API endpoints
Dependencies:
├── express (Web framework)
├── multer (File upload handling)
├── uploadFile() from storage.service.js
└── postModel (MongoDB Model)

Configuration:
├── Middleware: express.json() → Parse JSON request bodies
├── Upload: multer.memoryStorage() → Files stored in RAM (not disk)
└── Single file: upload.single("image") → Expects 1 file called "image"

ENDPOINT 1: POST /create-post
├── Input:
│   ├── req.file.buffer (image file)
│   └── req.body.caption (text)
├── Process:
│   ├── uploadFile(buffer) → Uploads to cloud, returns {url}
│   └── postModel.create({image: url, caption})
├── Output: Status 201 + {message, post object}
└── Error Handling: ⚠️ NONE - No try/catch

ENDPOINT 2: GET /posts
├── Input: None (fetches all posts)
├── Process: postModel.find() → No filter (gets all)
├── Output: Status 200 + {message, posts array}
└── Error Handling: ⚠️ NONE - No try/catch

Critical Details:
- multer.memoryStorage() = Files loaded into RAM (not written to disk)
- Good for small files, bad for large files
- upload.single("image") = Expects form-data with field name "image"
- No validation on caption length
- No error handling for upload failures
- No authentication/authorization

5. storage.service.js Breakdown
Line 1-2: Import ImageKit SDK
├── @imagekit/nodejs = Node.js client library for ImageKit API

Line 4-7: Initialize ImageKit
├── privateKey: process.env.IMAGEKIT_PRIVATE_KEY
├── Authentication for server-side uploads
├── ⚠️ IMPORTANT: Must be in .env file
└── This is PRIVATE - never expose in frontend code

Line 9-15: uploadFile(buffer, name) Function
├── Parameters:
│   ├── buffer = Image binary data from multer
│   ├── name = Filename (default: "upload.jpg")
│
├── Process:
│   ├── buffer.toString("base64") = Convert binary → Base64 string
│   │   └── ImageKit expects Base64 encoded file
│   │
│   └── imageKit.files.upload({...})
│       ├── file: base64 encoded image
│       └── fileName: name for the uploaded file
│
└── Returns: Promise with ImageKit response
    ├── Contains: fileId, url, filePath, etc.
    └── Used in app.js as: result.url


🔗 Complete Flow from User Upload to Database
1. FRONTEND sends multipart/form-data
   {
       file (image.jpg): [binary data],
       caption: "Amazing photo!"
   }

2. MULTER receives & processes
   req.file = {
       fieldname: "image",
       originalname: "photo.jpg",
       encoding: "7bit",
       mimetype: "image/jpeg",
       size: 245000,
       buffer: <Buffer 89 50 4e 47...>  ← Binary image data
   }

3. uploadFile(req.file.buffer) called
   ├── Converts buffer to Base64
   └── Sends to ImageKit API

4. ImageKit uploads to cloud
   ├── Stores image
   ├── Generates CDN URL
   └── Returns {url, fileId, ...}

5. app.js extracts URL
   image: "https://ik.imagekit.io/your-id/upload.jpg"

6. MongoDB saves
   {
       _id: ObjectId("..."),
       image: "https://ik.imagekit.io/your-id/upload.jpg",
       caption: "Amazing photo!",
       createdAt: ISODate("2024-01-15T10:30:00Z")
   }

7. API responds with 201
   {
       message: "Post created successfully",
       post: {
           _id: "...",
           image: "https://ik.imagekit.io/...",
           caption: "Amazing photo!",
           createdAt: "2024-01-15T10:30:00.000Z"
       }
   }

8. FRONTEND receives & displays
   <img src="https://ik.imagekit.io/..." alt="Amazing photo!" />

🔴 Remaining Issues************************
🔴 CRITICAL:
  ❌ No error handling in app.js routes (try/catch missing)
  ❌ No input validation (caption can be empty/huge)
  ❌ .env file not provided

🟠 HIGH PRIORITY:
  ❌ No authentication (anyone can upload)
  ❌ No authorization (can't delete own posts)
  ❌ No CORS configuration (frontend access blocked)

🟡 MEDIUM:
  ❌ No rate limiting (spam protection)
  ❌ No file size validation
  ❌ No file type validation (could upload non-images)
  ❌ Hardcoded filename "upload.jpg"
